# AI 视频生成 · 人物站位与走位调度 SOP

适用：Seedance 2.0 / 即梦 / 可灵 / Vidu 等文生·图生视频模型
用途：解决"人物站位摆不对、多人走位调度崩坏"两大核心问题

---

## 0. 一句话总纲

> **站位靠图像，走位靠首尾帧 + 续接，复杂调度先试长镜头、真崩了再拆镜头，文字只做轻量调度说明。**
> 能生成就抽卡筛选；**先把帅的长镜头 / in-camera 当主方案去试**，只有反复抽卡失败、确实不能稳定生成才别硬做——再改用剪辑、遮挡、反应镜头、分段合成。拆镜与后期是兜底，不是首选。

核心认知：**文字提示对"空间坐标"和"时间码"都是软约束（概率性），位置的硬约束必须交给图像。约束越想加得多，越说明该把负担从文字挪到图像上。**

> **生产原则（背下来）：图像负责空间，文字负责动作说明，分镜负责降低难度，抽卡负责筛选结果，剪辑负责模型做不到的部分。**

---

## 在本管线中的角色分工

本 SOP 是 `2-scene-layout`（可行性约束层）的方法论底座，但其中的知识跨越多个 skill。在 `1-script-to-draft → 2-scene-layout → 3-script-to-storyboard → 4-seedance-prompts` 这条链里：

- **2-scene-layout（本层）**：判难度、定**档位**、立**轴线**、列**关键帧需求与复用关系**、给**回避指令**、排**生成顺序**。即下面的 §1①–②④⑦⑨、§2、§3 档位表、§6、§8 的判断部分。
- **3-script-to-storyboard**：把关键帧的**具体画面**（景别/机位/光影/站位）落地，遵守本层的轴线与档位。即 §3.1 首尾帧图素材标准的执行。
- **4-seedance-prompts**：写**即梦文字提示词**。即 §5 提示词模板、§7.2 逐条提示词所展示的层。

下面的 §5 与 §7 完整示例保留**提示词写法**，是为了让你理解"文字层能承载什么、不能承载什么"（这决定本层怎么判档），**不是要本层去输出提示词**——提示词由 4-seedance-prompts 生成。

## 1. 总流程（封板版）

```
① 断能不能做（先筛掉模型做不了的调度）
        ↓
② 难站位先手动合成首帧（撞图像层难题时，用 PS / 区域控制 / 拼图解决）
        ↓
③ 图像锁站位（能首帧就首帧，能首尾帧就首尾帧）
        ↓
④ 尾帧续接（上一镜尾帧 = 下一镜首帧，连相邻镜头）
        ↓
⑤ 角色参考图（保跨镜头、切走又切回的人物一致性）
        ↓
⑥ 文字轻量补调度（方向 + 路径 + 不变量，3–5 个约束封顶）
        ↓
⑦ 分镜拆解（把复杂调度拆开，避开模型最弱场景）
        ↓
⑧ 抽卡筛选（同提示生成多条，挑站位最稳的，锁 seed）
        ↓
⑨ 做不到的交给剪辑 / 遮挡 / 合成
```

---

## 2. 第一步：先判"能不能做"（别硬塞给模型）

| 调度类型 | 可行性 | 处理 |
|---|---|---|
| 单人站立 / 转头 / 坐下 / 手部小动作 | ✅ 稳 | 文字或单首帧即可 |
| 两人靠近 / 擦肩 / 坐下 | 🟡 可尝试 | 首帧，必要时加尾帧 |
| 三人以上交叉走位 | 🔴 高风险 | 先用首/中/尾关键帧锁站位试长镜头，反复崩再拆 |
| 打斗 / 追逐 / 队形变化 | 🔴 极高风险 | 先试 in-camera 长镜头 + 关键帧；真崩再拆镜头 + 后期 |
| 要求精确踩点 / 精确路线 / 精确相对距离 | ❌ 别指望纯文字 | 手动合成关键帧锁住，仍优先一镜到底；不行再交给剪辑 |

## 3. 镜头四档分级表（核心生产清单）

| 档位 | 适用镜头 | 控制手段 |
|---|---|---|
| **纯文字** | 单人站立、轻微转头、坐着说话、手部小动作、反应特写 | 文字 3–5 个约束 |
| **首帧** | 多人对话、需明确左右关系、角色不能换位、场景结构重要 | 1 张构图准确的首帧图 → 图生视频 |
| **首帧 + 尾帧** | A 走到 B 旁、坐下、起身、进门、离开房间、靠近另一人 | 首帧锁起点，尾帧锁终点，模型插值 |
| **首帧 + 中间帧 + 尾帧** | 绕桌、追逐、打斗、多人交叉、队形变化、复杂调度 | 中间帧锁"穿越中点"，防漂移 |

**跨镜头续接**：上一条视频的**尾帧**直接作为下一条的**首帧** → 人物位置、服装、光线天然延续，剪辑不跳。

**跨镜头一致性**（切走又切回同一角色）：尾帧续接救不了，需 **角色参考图 / 锁 seed**。

### 3.1 首帧 / 尾帧图素材要求（做图时的硬标准）

- 人物必须全身、或至少半身清晰可见（被裁掉的部位模型会乱补）
- 人物之间不能互相遮挡关键身体轮廓
- 门、桌、窗、沙发等空间锚点要稳定、清楚
- 首帧和尾帧的镜头角度、光线、人物服装尽量一致
- **尾帧只改变必要站位**，不要同时改表情、构图、光线和道具（一次只动一个变量）

### 3.2 跨镜头交接规则

- 上一镜头尾帧 **=** 下一镜头首帧（位置、服装、光线直接续上）
- 若上一条视频"位置正确但表情不好"：**优先保留位置正确的尾帧**，表情后修
- 不要每个镜头重新设计站位，否则连续性会断

## 4. 场景设定模板（整场不变，贴进每条提示词）

```
场景空间：__（如：客厅，沙发在画面左侧，落地窗在右侧，门在后景中央）
人物外观锚点：A=__（服装+道具）；B=__；C=__
轴线：镜头始终从__方向拍，A 永远在 B 的画面__侧，不越轴、不反打。
```

> 外观锚点 + 轴线一旦定下，整场所有镜头复用，不要每镜重写不一致。

---

## 5. 提示词模板（文字层只写"最低必要信息"）

> 属 **4-seedance-prompts 层**。本层读它是为了判断"哪些信息文字扛不住、必须交给关键帧"，从而正确判档；本层不输出下面这种提示词。

```
场景空间：（复用上面的设定）
人物：A__，B__
动作：A 从__走向__，最后停在__；B__（写清谁动谁不动、路径方向）
镜头：景别__，平视，固定镜头（人物移动时镜头宁可不动），不反打，不改变左右关系
限制：不新增人物，不交换左右位置，不改变服装，不穿过__
首帧：__（起点画面）
尾帧：__（终点画面）
```

**减法原则——每条提示词只保留这 5 类信息**
1. 场景空间　2. 人物身份　3. 谁动、谁不动　4. 起点和终点　5. 1–3 条限制

**写法红线**
- ❌ 不写时间码（"0–1秒…1–3秒…"）。模型不按秒执行，时间码最多表达**先后顺序**，时长靠剪辑切。
- ❌ 不堆 10 个负面词。约束太多模型会丢重点，每条留最关键的 3–5 个。
- ❌ 不写剧情（"两人对峙"）。只写空间关系（"A 在左前景侧身，B 在右后景"）。
- ✅ 人物移动时镜头尽量固定 / 缓慢横移 / 跟拍其一，不要环绕+推镜+焦点转移同时上。

## 6. 失败处理流程（抽卡是工作流，不是失败）

```
生成多条（同首尾帧 + 同简洁提示词）
        ↓
挑站位最稳的一条 → 锁定 seed
        ↓
还不稳？按优先级处理（不要继续加词硬怼）：
  1. 换 / 补尾帧图
  2. 缩短镜头时长
  3. 拆成更短的镜头
  4. 改用反应镜头 / 遮挡，跳过难走位的过程
  5. 分开拍单人元素 → 后期合成
```

> 记住："最难的走位，最好的解法常常是不拍它。" 用景别和遮挡藏掉难点，比在单条提示里死磕命中率高得多。

## 7. 完整示例：三人客厅对峙

> 端到端示例：§7 表格 + §7.1 关键帧清单是**本层**的产物形态（档位、关键帧需求与复用）；§7.2 逐条提示词是 **4-seedance-prompts 层**的产物，放这里是为了看清整条链怎么落地。

**设定**：客厅，沙发在左、落地窗在右、门在后景中央。A=黑色外套手拿文件，B=白衬衫，C=灰色帽子。镜头正面拍，A 永远在 B 的画面左侧。

| 镜头 | 内容 | 档位 | 理由 |
|---|---|---|---|
| 1 | A 推门进来停在门口 | 仅首帧 | 单人小幅移动 |
| 2 | A 走到沙发前，B 抬头 | 首+尾帧 | 有明确走位终点 |
| 3 | C 从窗边转身走到 A、B 之间 | 首+中+尾帧 | 三人交叉，中间帧防漂移 |
| 4 | 三人对峙，B 起身 | 首+尾帧 | 坐→站是位置突变 |
| 5 | A 脸部反应特写 | 纯文字 | 单人无走位 |

续接：镜头1尾帧→镜头2首帧；镜头2尾帧→镜头3首帧。
注意：镜头3命中率最低，可拆成"C 转身(单人)" +"切到三人已就位的中景"，用剪辑跳过穿越过程。

### 7.1 需要的关键帧图清单（先做图，再生成）

| 关键帧 | 画面 | 来源 |
|---|---|---|
| K1 | A 在后景门口，手扶门，客厅全貌 | 文生图 / 合成 |
| K2 | A 停在沙发前画面左侧面向 B，B 坐沙发抬头 | = 镜头2尾帧（镜头3首帧复用） |
| K3 | C 走到 A、B 之间（画面中央），三人成行 | **建议手动合成**（三人站位文生图难控） |
| K4 | B 已起身站在沙发前，三人对峙 | 合成 / 在 K3 基础上改 B 姿态 |

> K1=镜头1尾帧、镜头2首帧；K2=镜头2尾帧、镜头3首帧；K3=镜头3中/尾帧、镜头4首帧；K4=镜头4尾帧。一张图多镜复用 = 续接落地。

### 7.2 五镜逐条提示词

**镜头1（仅首帧 K1）**
```
场景空间：客厅，沙发在左、落地窗在右、门在后景中央。
人物：A 黑色外套手拿文件。
动作：A 推门进入，停在后景门口，面向客厅内。
镜头：全景，平视，固定镜头。
限制：不新增人物，门窗位置不变。
首帧：K1（A 在门口手扶门）。
```

**镜头2（首帧 K1 + 尾帧 K2）**
```
场景空间：客厅，沙发在画面左侧，落地窗在右侧，门在后景中央。
人物：A 黑色外套手拿文件，B 白衬衫坐在沙发。
动作：A 从后景门口走向画面左侧的沙发前停下，面向 B；B 坐着不动，只抬头看向 A。
镜头：中全景，平视，固定镜头，不反打，不改变左右关系。
限制：不新增人物，不交换 A 和 B 的位置，A 不穿过沙发，背景门窗位置不变。
首帧：K1。　尾帧：K2。
```

**镜头3（首帧 K2 + 中间帧 + 尾帧 K3）— 全场最难，优先考虑拆**
```
场景空间：同上。
人物：A 黑色外套（画面左），B 白衬衫坐沙发，C 灰色帽子。
动作：C 从画面右侧窗边转身，走到 A 与 B 之间（画面中央）停下；A、B 不移动。
镜头：中景，平视，缓慢横移跟随 C，不反打。
限制：不交换三人左右顺序，C 不穿过茶几，不新增人物。
首帧：K2。　中间帧：C 行至画面中央。　尾帧：K3。
```
> 抽卡若反复崩 → 拆成「3a：C 单人转身起步（仅首帧）」+「3b：切到 K3 三人已就位中景（仅首帧）」，剪辑跳过穿越。

**镜头4（首帧 K3 + 尾帧 K4）**
```
场景空间：同上。
人物：A、B、C 三人就位，B 白衬衫坐沙发。
动作：B 从坐姿起身，站到沙发前；A、C 不移动。
镜头：中景，平视，固定镜头，不反打。
限制：不交换三人位置，不新增人物，服装不变。
首帧：K3。　尾帧：K4（B 已站起）。
```

**镜头5（纯文字）**
```
人物：A 黑色外套。
动作：A 面向画面外，神情凝重，轻微转头。
镜头：近景特写，平视，固定镜头。
限制：单人，无走位，背景虚化。
```

## 8. 速记卡

- 站位问题 → **图像**（首帧 / 手动合成首帧）
- 走位问题 → **首尾帧 + 尾帧续接**
- 复杂调度 → **拆镜头**
- 跨镜头一致 → **角色参考图 / 锁 seed**
- 文字 → **只做轻量调度说明（方向+路径+不变量，≤5 约束，无时间码）**
- 不稳 → **抽卡筛选**；做不到 → **剪辑 / 遮挡 / 合成**
