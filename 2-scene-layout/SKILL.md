# 剧本调度分析（AI 可行性约束层）

把已经拆好的镜头列表，过一遍"AI 视频模型能不能稳定生成"，在 storyboard 动手设计之前先把约束立起来。核心解决两件事：**人物站位摆不对** 和 **多人走位调度崩坏**。

## 在管线里的位置

```
1-script-to-draft → 【2-scene-layout ← 你在这里】 → 3-script-to-storyboard → 4-seedance-prompts
```

- **上游 1-script-to-draft** 已经锁定镜头数量和顺序（"只设计每个镜头，不增减数量"）。本层不改镜头结构，只在这条已锁定的镜头序列上做可行性判断——这正是本层能放在 storyboard 之前的前提。
- **本层的产物是一份约束 brief**，交给 storyboard。让可行性成为**设计输入**：storyboard 据此锁轴线、对重灾区选回避性景别/遮挡、给关键帧镜头设计可落地的画面——而不是设计完一套漂亮分镜再被审计推翻、回炉重做。
- **本层刻意不做两件事**：① 不写即梦提示词（那是 4-seedance-prompts 的活，避免两套格式打架）；② 不定关键帧图的**具体画面**（确切景别/机位/光影要等 storyboard 设计完才能落地，本层只定"哪些镜要关键帧 + 复用关系 + 站位意图"）。

## 核心原则（先内化）

> **图像负责空间，文字负责动作说明，分镜负责降低难度，抽卡负责筛选结果，剪辑负责模型做不到的部分。**

文字提示对"空间坐标"和"时间码"都是**软约束（概率性）**；位置的硬约束必须交给**图像（首帧 / 尾帧 / 关键帧）**。约束越想往提示词里加得多，越说明该把负担从文字挪到图像上。详见 `references/SOP.md`。

> **野心优先（同等重要，别只记上一条）**：「降低难度」不等于「一上来就拆镜」。高价值镜头先按**长镜头 / in-camera 设计成主方案**，真去抽卡试；**拆镜是抽卡反复失败之后的标注兜底，不是默认**。最帅的镜头往往就得一镜到底——别因为「可能难」就预先把它切碎。降难度的手段优先级是：**先 in-camera 长镜头 → 用关键帧锁站位 → 缩短时长 → 才考虑拆镜**。

## 输入

来自 1-script-to-draft 的草稿：导演讲戏稿（含用户批注）、场景视觉基调块、完整镜头列表。**用户批注优先。** 本层据此判断难度并立约束，不改动镜头数量与顺序。

若用户只给了原始剧本、还没有镜头草稿：先提示走 `1-script-to-draft` 拆出镜头列表再回到本层；用户坚持时可直接基于剧本快速判难度结构，但提醒其结论会在草稿成型后更准。

## 工作流程

收到镜头列表后，按顺序执行：

### 1. 先判整场难度结构
通读镜头列表，先快速分出**省事镜**和**重灾区**（多人同框、反打/主观视角越轴、动态动作如跳/推/抢/掷），把精力压在重灾区。确认**角色数量与群演规模**、画面里包围/人群要多"实"。这一步决定后面把多少约束力气花在哪。

### 2. 建全局约束
- **场景空间**：地点、光源、关键道具位置（整场不变，写进 brief 供 storyboard 与下游沿用）。
- **轴线**：定下谁在画面左 / 右，所有机位锁在连线同一侧，不反打、不越轴。**这条是给 storyboard 的硬约束**——storyboard 设计机位时必须遵守，而不是事后被本层抓越轴。
- **角色清单**：列出所有**需要一致性**的角色，每人一句外观锚点（服装 + 道具 + 神态）。提醒用户为每个角色做**角色参考图**（锁脸锁服装）；切走又切回的角色靠参考图 / 锁 seed 保持一致。
- **群演**：不需要一致性的人群默认按**剪影 / 背景虚焦 / 暗部**处理。人一多崩坏率指数上升，一镜只渲核心人物 + 暗部剪影群演；需要"包围气势"时用**分层合成**叠出人多。

### 3. 逐镜判档（四档分级）

**判档前先问一句：这镜能不能一镜到底？** 单 / 双主体的运动镜头（走、跑、起跳、踹、走向、靠近、碰撞、转身掷物）默认按**长镜头**设计，用首 / 尾帧锁住起止站位即可——档位（首帧 / 首尾帧 / 关键帧）是给长镜头**锁站位**的工具，**不是把它切成多镜的理由**。只有多人交叉走位真的反复抽卡崩、或一条里塞了不相容景别，才考虑拆。

| 档位 | 适用 | 控制手段 |
|---|---|---|
| 纯文字 | 单人站立 / 转头 / 坐着说话 / 反应特写 | 文字 ≤5 类信息 |
| 首帧 | 多人对话、需明确左右、角色不能换位、场景结构重要 | 1 张构图准确首帧 → 图生视频 |
| 首+尾帧 | 走到某处、坐下、起身、进门、靠近他人 | 首帧锁起点、尾帧锁终点 |
| 首+中+尾帧 | 绕行、追逐、打斗、多人交叉、碰撞、队形变化 | 中间帧锁穿越中点 |

同时标出**风险等级**与**重灾区**：多人同框、反打 / 主观视角（越轴）、动态动作。档位标注会随 brief 进入 storyboard 的头部格式行，一路带到 4-seedance-prompts。

### 4. 关键帧需求与复用关系
为需要图像约束的镜头标出**需要关键帧（KF）**及**复用关系**：上一镜尾帧 = 下一镜首帧（续接落地），一张图尽量服务多个镜头。

**本步只定"需求 + 复用 + 站位意图"，不定画面细节**——每张 KF 的确切景别/机位/光影/构图由 storyboard 设计该镜时落地，本层给的是"这里需要一张首帧锁住 A 在左 B 在右"这一级的意图。

### 5. 难度回避（长镜头优先）
- **野心优先于回避**：默认把帅的版本（长镜头 / in-camera）当主方案去抽卡试；下面这条「做不到就别硬做」只在**真去抽卡、反复失败之后**才启用。
- **做不到的（真做不到时）别硬做**：反复崩的最难走位，最好的解法常常是不拍它，用景别和遮挡藏掉难点。把这类**回避指令**写进 brief（如"此多人交叉镜若抽卡反复崩，退到近景 + 前景遮挡只拍局部"），但标明它是**兜底、不是首选**。
- **抽卡是工作流**：同首尾帧 + 同简洁提示词生成多条，挑站位最稳的、锁 seed。不稳时按"换尾帧 → 缩短镜头 → 拆镜头 → 反应镜头 / 遮挡 → 分段合成"处理，**不要继续加词硬怼**（这条给下游生成阶段执行）。

### 拆生成 vs 结构返修（重要区分）

本层会"拆镜头"，但要分清两种拆，别混：

- **拆生成单元（本层自己做，不改分镜结构）**：一个分镜镜头在**生成时**拆成多条 AI 片段、再靠剪辑拼起来（如飞刀镜拆 5–6 条）。镜号不变、分镜结构不变，只是生产层的切分——写进 B 表的"处理要点"，不回送 draft。
- **结构返修（回送 draft，本层不硬吞）**：某个分镜镜头本身就违反单镜原则——一个镜头里塞进了本该拆成多镜的多人走位或跨景别内容，靠档位 / 关键帧 / 回避都救不了。这是 draft 层的结构问题，本层把它列进「结构返修建议」回送 `1-script-to-draft`，由它拆成多个镜头、更新列表后再走一遍；本层只对单镜可行的内容立约束。

判据：能靠"生成时拆片段 + 剪辑"解决的 → 拆生成；必须改变**分镜镜号/数量**才合理的 → 结构返修。

## 输出格式

产出一份**约束 brief**（建议存为 .md 文件并用 present_files 展示），交给 `3-script-to-storyboard`，含：

A. **全局约束**（场景 / 轴线 / 角色锚点表 / 群演策略）
B. **逐镜档位与风险表**（镜号 | 内容摘要 | 档位 | 风险等级 | 主方案 | 兜底）
　 - **主方案**默认是长镜头 / in-camera（标"一镜到底 / 跟拍"等）；
　 - **兜底**仅高危镜才填（拆镜 / 含蓄回避），并注明"仅抽卡反复失败时启用"，不是默认执行项。
C. **关键帧需求与复用关系**（哪些镜要 KF、续接链；**只到站位意图，不含画面细节**）
D. **结构返修建议（如有）**——列出需要 draft 拆分 / 调整的镜号与原因（仅限结构性不可行的镜头）；无则不出此节。

**不输出逐镜即梦提示词**——那是 4-seedance-prompts 的职责。
**不输出关键帧画面细节**——那由 storyboard 设计时落地。

完整方法论与档位/关键帧判断模板见 `references/SOP.md`；完整范例见 `references/example-dock-night.md`。

## 下游接口

**先判有无结构返修**：若 brief 含「结构返修建议」（D 节），先回到 `1-script-to-draft` 按建议拆分镜头、更新镜头列表，再重跑本层——结构定稳后才进 storyboard。其余情况：把约束 brief 连同原镜头列表一起传给 `3-script-to-storyboard`。storyboard 设计每镜时须遵守：

1. **轴线** —— 不越轴、不擅自反打。
2. **逐镜档位** —— 在头部格式行末尾带上本镜档位标注，一路传给 4-seedance-prompts。
3. **重灾区回避指令** —— 按 brief 指定的回避手段（回避性景别 / 遮挡 / 机位）设计。
4. **关键帧落地** —— 为标了 KF 的镜头，把首帧 / 尾帧各自的画面（景别 / 机位 / 光影 / 站位）在画面内容里写具体，供导演预先生成关键帧图、供 4-seedance-prompts 引用。

## 开场建议

如果用户只说"分析一下"而未提供镜头草稿，先索取 1-script-to-draft 的镜头列表（或剧本）。拿到后，**先报整场难度结构**（哪些镜头省事、哪些是重灾区，精力压在重灾区），确认**角色数量与群演规模**、**画面里包围 / 人群要多"实"**，再展开完整约束 brief。
